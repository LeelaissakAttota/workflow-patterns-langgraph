## 📌 Project Overview

This project demonstrates **three fundamental workflow patterns** that 
form the architectural foundation of every enterprise AI system — 
implemented using **LangGraph** and **GPT-4o-mini**.

These patterns transform individual AI models into **sophisticated, 
coordinated multi-agent systems** capable of handling complex, 
real-world tasks intelligently.

## 🔗 Pattern 1 — Prompt Chaining (Sequential)

### Use Case: Job Application Assistant

```
Job Description Input
        │
        ▼
┌───────────────────────────┐
│  generate_resume_summary  │
│  GPT-4o-mini              │
│  → resume_summary         │
└───────────┬───────────────┘
            │
            ▼
┌───────────────────────────┐
│  generate_cover_letter    │
│  GPT-4o-mini              │
│  Uses resume_summary +    │
│  job_description          │
│  → cover_letter           │
└───────────────────────────┘
```

**State:**
```python
class ChainState(TypedDict):
    job_description: str   # Input
    resume_summary: str    # Generated by Agent 1
    cover_letter: str      # Generated by Agent 2
```

**Graph:**
```python
workflow.add_node("generate_resume_summary", generate_resume_summary)
workflow.add_node("generate_cover_letter", generate_cover_letter)
workflow.set_entry_point("generate_resume_summary")
workflow.add_edge("generate_resume_summary", "generate_cover_letter")
workflow.set_finish_point("generate_cover_letter")
```

**Test Input:**
```python
{"job_description": "We are looking for a data scientist with ML, NLP, and Python experience..."}
```

## 🔀 Pattern 2 — Routing (Intent-Based)

### Use Case A: Task Classifier (Summarize vs Translate)

```
User Input
    │
    ▼
┌──────────────┐
│ router_node  │ ← LLM classifies intent using bind_tools()
│ (Classifier) │
└──────┬───────┘
       │
   ┌───┴───┐
   ▼       ▼
summ   translate
arize   _node
_node
```

**Tool-Based Router:**
```python
class Router(BaseModel):
    role: str = Field(description="Output 'summarize' or 'translate'")

llm_router = llm.bind_tools([Router])

def router_node(state):
    response = llm_router.invoke(routing_prompt)
    return {**state, "task_type": response.tool_calls[0]['args']['role']}
```

**Conditional Routing:**
```python
workflow.add_conditional_edges("router", router, {
    "summarize": "summarize",
    "translate": "translate"
})
```

### Use Case B: Multi-Service Router (Exercise)

4-way routing for delivery services:

```
User Input → Router → ride_hailing_call
                   → restaurant_order
                   → groceries
                   → default_handler
```

**4 Specialized Nodes:**

| Node | Intent | Example Input |
|---|---|---|
| `ride_hailing_call` | Ride booking | "I need a ride to the airport at 3pm" |
| `restaurant_order` | Food ordering | "Order 2 large pepperoni pizzas" |
| `groceries` | Grocery delivery | "I need milk, bread, eggs" |
| `default_handler` | Unclassified | "What's the weather today?" |

**Router with Fallback:**
```python
def router_node(state):
    response = llm_router.invoke(state['user_input'])
    if response.tool_calls:
        return {**state, "task_type": response.tool_calls[0]['args']['role']}
    else:
        return {**state, "task_type": "default_handler"}  # Fallback
```

## ⚡ Pattern 3 — Parallelization (Concurrent Execution)

### Use Case: Multilingual Translation Assistant

```
"Good morning! I hope you have a wonderful day."
                        │
          ┌─────────────┼─────────────┐
          │             │             │
          ▼             ▼             ▼
   translate_     translate_    translate_
   french         spanish       japanese
   (parallel)     (parallel)    (parallel)
          │             │             │
          └─────────────┼─────────────┘
                        │
                        ▼
                  ┌──────────┐
                  │aggregator│
                  │combines  │
                  │all 3     │
                  └──────────┘
                        │
                        ▼
                  combined_output
```

**State:**
```python
class State(TypedDict):
    text: str              # Original English
    french: str            # French translation
    spanish: str           # Spanish translation
    japanese: str          # Japanese translation
    combined_output: str   # All merged
```

**Parallel Fan-out from START:**
```python
# All 3 translation nodes start simultaneously
graph.add_edge(START, "translate_french")
graph.add_edge(START, "translate_spanish")
graph.add_edge(START, "translate_japanese")

# All merge at aggregator
graph.add_edge("translate_french", "aggregator")
graph.add_edge("translate_spanish", "aggregator")
graph.add_edge("translate_japanese", "aggregator")
graph.add_edge("aggregator", END)
```

## 🔑 LangGraph Concepts Mastered

| Concept | Implementation |
|---|---|
| `StateGraph` | Shared TypedDict state across nodes |
| `set_entry_point()` | Define workflow start node |
| `add_edge()` | Sequential connections |
| `add_conditional_edges()` | Dynamic routing based on state |
| `set_finish_point()` | Multiple valid end nodes |
| `bind_tools()` | Pydantic tool for LLM classification |
| Parallel fan-out | Multiple `add_edge(START, node)` |
| State immutability | `{**state, "key": value}` pattern |
| `llm.invoke()` | Direct LLM calls within nodes |
| Graph visualization | `draw_png()` |

## 📊 Pattern Comparison

| Pattern | Structure | Best For | Example |
|---|---|---|---|
| **Prompt Chaining** | Sequential → → → | Step-by-step refinement | Resume → Cover Letter |
| **Routing** | Branch (one path) | Intent classification | Summarize vs Translate |
| **Parallelization** | Fan-out → merge | Independent subtasks | Multi-language translation |
